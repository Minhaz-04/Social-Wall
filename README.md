# Social-Wall
Backend File Names (Django)
These handle server-side logic, database, and APIs:
- manage.py – entry point to run the Django project.
- project_name/settings.py – configuration (database, installed apps, middleware).
- project_name/urls.py – URL routing.
- app_name/views.py – request handling, business logic.
- app_name/models.py – database schema definitions.
- app_name/admin.py – admin panel configuration.
- app_name/forms.py – form handling (optional).
- app_name/api/ – if using Django REST Framework, contains serializers and API views.
